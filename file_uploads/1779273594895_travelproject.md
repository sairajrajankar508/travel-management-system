# 📌 Product Requirements Document (PRD)
## Project: Travel Management System (TMS)

---

## 🎯 1. Objective

Build a full-stack Travel Management System that automates the entire lifecycle of employee travel:
- Request creation
- Multi-level approvals
- Expense tracking
- Reimbursement processing
- Reporting

The system must follow clean architecture and production-level standards.

---

## 🧱 2. Tech Stack

### Frontend
- React (Vite)
- JavaScript (NOT TypeScript)
- Redux Toolkit + RTK Query
- Vanilla CSS
- React Router (useNavigate, Outlet)
- React Hook Form + Zod Resolver
- npm-hot-toast

### Backend
- Java (Spring Boot)
- REST APIs
- MySQL Database

---

## 👥 3. User Roles

### 1. Admin
- Create/manage users
- Assign roles (Employee, Manager, Finance)
- Define travel policies

### 2. Employee
- Create travel requests
- Save draft / submit
- Track request status
- Add expenses

### 3. Manager
- Approve/Reject requests
- Add comments

### 4. Finance
- Final approval
- Process reimbursements

---

## 🔄 4. Core Workflow

1. Employee creates request
2. Request goes to Manager
3. Manager approves/rejects
4. If approved → goes to Finance
5. Finance approves
6. Employee submits expenses
7. Reimbursement processed

---

## 🧩 5. Features

### 🧾 Travel Request
- Destination
- Travel dates
- Purpose
- Estimated budget
- Draft + Submit

### ✅ Approval System
- Multi-level approval
- Status tracking:
  - Draft
  - Pending Manager
  - Pending Finance
  - Approved
  - Rejected

### 📏 Policy Enforcement
- Budget limits
- Rule validation
- Violation flagging

### 🧳 Itinerary
- Travel segments
- Stay details
- Timeline view

### 💰 Expense Management
- Add expense entries
- Categories:
  - Food
  - Stay
  - Transport
- Upload proof (simulate file storage)

### 📊 Reporting
- Employee history
- Department spend
- Total cost
